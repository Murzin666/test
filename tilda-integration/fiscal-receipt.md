# Налоговые атрибуты чека (ФФД) по товарам

Раз Tilda не может передавать разные `taxRate`/`type`/`mode` по каждому
товару (см. переписку — эти настройки в форме Tilda не привязаны ни к
какому передаваемому параметру), сервер сам собирает чек для банка,
сопоставляя товары из корзины Tilda (по **названию**, так как отдельного
поля SKU в массиве "Товары" у Tilda нет) со своим каталогом.

## Как это работает

1. Tilda присылает состав корзины в поле `products` (JSON-массив с
   `name`/`quantity`/`price` на каждый товар).
2. Сервер для каждого товара ищет запись в своей таблице `products` по
   **точному совпадению названия**.
3. Если нашёл — берёт `taxRate`/`type`/`mode` оттуда.
4. Если не нашёл — берёт запасные атрибуты по умолчанию для всей точки
   (`default_tax_rate`/`default_item_type`/`default_calc_mode`, задаются при
   добавлении точки) и пишет предупреждение в лог сервера — стоит проверять
   логи после запуска нового товара в продажу, чтобы вовремя настроить его.
5. Если у точки не указан ИНН — чек вообще не отправляется банку (банк
   требует ИНН в каждом запросе с чеком), сервер продолжит работать, просто
   без блока `receipt` — банк применит свои настройки терминала по умолчанию,
   как было раньше.

**Важно:** название товара должно **точно совпадать посимвольно** с тем, что
указано в карточке товара на Tilda (включая регистр и пробелы) — это и есть
ключ поиска в каталоге.

## Команды

```bash
# Добавить/обновить товар
python3 manage_tenants.py product add shop1 "Груши" --tax-rate 2 --type 1 --mode 4

# Список товаров точки
python3 manage_tenants.py product list shop1

# Удалить товар из каталога (снова начнёт использовать дефолты точки)
python3 manage_tenants.py product remove shop1 "Груши"

# Посмотреть ИНН/дефолты/товары точки целиком
python3 manage_tenants.py show shop1
```

На Railway — то же самое через `railway run python manage_tenants.py product add ...`.

## Коды `taxRate` (ставка НДС)

Значения для ОФД Orange Data (целые числа):

| Код | Значение |
|---|---|
| 1 | НДС 18% |
| 2 | НДС 10% |
| 3 | НДС расч. 18/118 |
| 4 | НДС расч. 10/110 |
| 5 | НДС 0% |
| 6 | НДС не облагается |

Для ОФД БИФИТ Онлайн — строки: `WITHOUT_VAT`, `VAT_0`, `VAT_10`, `VAT_20`,
`VAT_110`, `VAT_120`. Сервер передаёт значение как есть (число или строку),
уточните у банка, какой ОФД используется на вашем терминале.

## Коды `type` (признак предмета расчёта)

| Код | Значение |
|---|---|
| 1 | Товар |
| 2 | Подакцизный товар |
| 3 | Работа |
| 4 | Услуга |
| 10 | Платёж |
| 11 | Агентское вознаграждение (если продаёте как агент/комиссионер) |
| 13 | Иной предмет расчёта |

(Полный список — в исходном руководстве банка, раздел про операцию
«Создание заказа».)

## Коды `mode` (признак способа расчёта)

| Код | Значение |
|---|---|
| 1 | Предоплата 100% |
| 2 | Частичная предоплата |
| 3 | Аванс |
| 4 | Полный расчёт |
| 5 | Частичный расчёт и кредит |
| 6 | Передача в кредит |
| 7 | Оплата кредита |

## Признак агента

Отдельного поля для этого у банка нет — если товар продаётся по агентской
схеме, для него просто указывается `--type 11` («Агентское вознаграждение»)
в каталоге товаров этой позиции.

## Обязательные поля точки

- `--inn` — ИНН, без него банк не примет чек (не имеет дефолта).
- `--tax-system-code` — код системы налогообложения (1=ОСН, 2=УСН доходы,
  3=УСН доходы-расходы, 4=ЕНВД, 5=ЕСН, 6=Патент).
