# Интеграция с Tilda

- **`universal-payment-system.md`** — настройка официальной Универсальной
  платёжной системы Tilda (как заполнить форму, какие поля и правила подписи
  использовать). Начните отсюда.
- **`multi-tenant.md`** — как сервер обслуживает несколько торговых точек:
  Railway Volume, ключ шифрования, команды `manage_tenants.py` для
  добавления/просмотра/удаления точек.
- **`fiscal-receipt.md`** — как задать НДС/признак предмета расчёта/признак
  способа расчёта по каждому товару (Tilda это не передаёт, атрибуты хранятся
  в каталоге товаров на сервере и сопоставляются по названию).

Более ранний вариант через перехват формы корзины кодом в блоке T123 был
заменён официальным протоколом Tilda и удалён из проекта — он был менее
надёжным (не основан на документированном API, не сообщал об оплате в CRM
Tilda автоматически).
